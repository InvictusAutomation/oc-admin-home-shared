# OpenClaw AI Team - 团队配置

## 团队架构

| 角色 | 名称 | 功能 | 状态 |
|------|------|------|------|
| Leader | capi | 总协调、任务分发、结果汇总 | 就绪 |
| Researcher | nova | 网页搜索、内容抓取、信息收集 | 就绪 |
| Coder | bolt | 编程开发、代码调试 | 就绪 |
| Analyst | iris | 数据分析、报告生成 | 就绪 |
| Writer | echo | 文档撰写、内容创作 | 就绪 |
| QA | lens | 测试验证、质量检查 | 就绪 |

## 协作协议

### 任务流程
1. Leader 接收任务 → 分析复杂度
2. 简单任务 → 直接分配给合适 Agent
3. 复杂任务 → 拆分为子任务 → 并行执行
4. 依赖关系 → 前置完成 → 自动解锁下游
5. 结果汇总 → Leader 统一交付

### 沟通机制
- 任务状态：pending / in_progress / completed / blocked
- 信息同步：通过 sessions 机制
- 冲突解决：Leader 仲裁

## 使用方式

### 启动团队
```
使用 sessions_spawn 启动子任务
```

### 任务分配示例
- "写一个爬虫" → 分配给 bolt
- "分析这个网站" → 分配给 nova
- "生成报告" → 分配给 iris + echo
- "测试这个功能" → 分配给 lens

## 纪律守则

1. **不越权** - 只做职责范围内的事
2. **不重复** - 已有答案不重复搜索
3. **及时汇报** - 遇到问题立即上报
4. **结果导向** - 最终交付可用成果
5. **安全第一** - 不执行危险命令
